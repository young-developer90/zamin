## Getting Started

### Installation

Build from source (requires Rust 1.80+):

```bash
git clone https://github.com/young-developer90/lion.git
cd lion
cargo build --release
```

The binary will be at `target/release/lion` (or `target\release\lion.exe` on Windows).

### Running Code

```bash
lion run hello.lion          # Run a script file
lion repl                    # Interactive REPL session
lion run --disassemble file  # Show generated bytecode
lion fmt file                # Format source code
lion test path/              # Run test suite
```

Inside the REPL:

```
Lion REPL v0.1.0
Type 'exit' to quit, 'help' for help.
lion> print("hello world")
hello world
=> nil
lion> 1 + 2
=> 3
lion> let x = 10
=> 10
```

### Your First Program

Create `hello.lion`:

```lion
print("Hello, Lion!");
```

Run it:

```bash
lion run hello.lion
```

Comments are ignored by the interpreter and are used to document your code.

```lion
// This is a single-line comment

/*
 * This is a multi-line comment.
 * It can span multiple lines.
 */
```

## Variables

Variables are declared with `let` for mutable values and `const` for constants.

```lion
let x = 10;         // mutable variable
x = 20;             // reassign works
x += 5;             // add-assign (also -=, *=, /=)

const y = 42;       // immutable — cannot be reassigned

let z;              // declared but uninitialized (defaults to nil)
z = "hello";        // now it holds a string
```

### Type Annotations

Optional type annotations let you document intent. They don't enforce at runtime but improve readability.

```lion
let name: String = "Lion";
let count: Int = 5;
let pi: Float = 3.14;
let flag: Bool = true;
```

### Scope

Variables are block-scoped. Inner blocks can see outer variables.

```lion
let x = 1;
{
    let y = 2;
    print(x, y);    // 1 2
}
print(x);           // 1
// print(y);        // error — y is out of scope
```

## Data Types

Lion is dynamically typed. The runtime type of any value can be checked with `type()`.

| Type    | Examples                      | Notes                        |
|---------|-------------------------------|------------------------------|
| Int     | `42`, `-17`, `0`              | 64-bit signed integer        |
| UInt    | `(as u64 literal)`            | Unsigned integer             |
| Float   | `3.14`, `-0.5`, `2.0`        | 64-bit floating point        |
| String  | `"hello"`, `"""multi-line"""` | Immutable text               |
| Bool    | `true`, `false`               | Boolean values               |
| Nil     | `nil`                         | Represents absence of value  |
| List    | `[1, 2, 3]`                   | Ordered, mutable collection  |
| Dict    | `{"a": 1, "b": 2}`            | Key-value map                |
| Set     | `{1, 2, 3}`                   | Unique elements              |
| Tuple   | `(1, "hello", 3.0)`           | Fixed-size heterogeneous     |
| Function| `func(x) { return x * 2 }`    | Callable values              |
| Lambda  | `\|x\| x * 2`                 | Anonymous function shorthand |

```lion
print(42);         // Int
print(3.14);       // Float
print("hello");    // String
print(true);       // Bool
print(nil);        // Nil
print([1, 2, 3]);  // List
print({"a": 1});   // Dict
```

## Operators

### Arithmetic

```lion
1 + 2       // add -> 3
5 - 3       // subtract -> 2
4 * 3       // multiply -> 12
10 / 3      // divide -> 3.333...
10 // 3     // integer division -> 3
10 % 3      // modulo -> 1
2 ** 10     // power -> 1024
```

### Comparison

```lion
1 == 1      // true
1 != 2      // true
2 < 3       // true
2 > 3       // false
2 <= 2      // true
3 >= 2      // true
```

### Containment (`in`)

The `in` operator checks membership across all collection types:

```lion
1 in [1, 2, 3]       // true  — list membership
4 in [1, 2, 3]       // false
"a" in {"a": 1}      // true  — dict key membership
2 in {1, 2, 3}       // true  — set membership
"el" in "hello"      // true  — substring check
"x" in "hello"       // false
3 in 0..10           // true  — range containment
15 in 0..10          // false
```

### Chained Comparisons

Comparisons can be chained naturally:

```lion
0 < 5 < 10           // true  — equivalent to (0 < 5) and (5 < 10)
0 < 10 < 5           // false
1 <= x <= 10         // check if x is in range [1, 10]
```

Only `<`, `>`, `<=`, `>=` can be chained. `==`, `!=`, and `in` do not chain.

### Logical

```lion
true and false  // false
true or false   // true
not true        // false
```

### String Concatenation

```lion
"hello" + " world"   // "hello world"
"hello" .. " world"  // also concatenation (alternative syntax)
```

## Strings and Interpolation

### String Interpolation

Use `f""` strings to embed expressions directly:

```lion
let name = "Lion";
print(f"Hello, {name}!");        // Hello, Lion!
print(f"2 + 2 = {2 + 2}");       // 2 + 2 = 4
print(f"pi ≈ {3.14159}");        // pi ≈ 3.14159
```

Any valid expression can go inside `{}`:

```lion
let items = [1, 2, 3];
print(f"length: {items.len()}, first: {items[0]}");
// length: 3, first: 1
```

### Multi-line Strings

Use triple quotes for multi-line strings:

```lion
let s = """this is
a multi-line
string""";
print(s);
```

### String Methods

Strings have built-in methods accessible through the `string` module:

```lion
string.upper("hello")     // "HELLO"
string.lower("WORLD")     // "world"
string.trim("  hi  ")     // "hi"
string.split("a,b,c", ",")// ["a", "b", "c"]
string.reverse("abc")     // "cba"
string.repeat("ab", 3)    // "ababab"
string.contains("hello", "el") // true
string.replace("hi there", "hi", "hello") // "hello there"
string.substring("hello", 1, 4) // "ell"
```

## Control Flow

### If / Elif / Else

```lion
let x = 10;

if (x > 5) {
    print("x is big");
} elif (x < 0) {
    print("x is negative");
} else {
    print("x is small");
}
```

Parentheses around conditions are optional:

```lion
if x > 5 {
    print("big");
}
```

### While Loop

```lion
let i = 0;
while (i < 5) {
    print(i);
    i = i + 1;
}
// prints 0, 1, 2, 3, 4
```

### For Loop

Iterate over ranges and collections:

```lion
for i in 0..4 {
    print(i);   // prints 0, 1, 2, 3
}

let items = [10, 20, 30];
for item in items {
    print(item);
}   // prints 10, 20, 30

let dict = {"a": 1, "b": 2};
for key in dict {
    print(key);   // prints "a", "b" (iteration over keys)
}
```

### Ternary Expression

```lion
let max = a > b ? a : b;
let result = x >= 0 ? "positive" : "negative";

// Nested ternaries
let label = x > 0 ? "pos" : x < 0 ? "neg" : "zero";
```

### Break and Continue

```lion
for i in 0..10 {
    if i == 3 {
        continue;   // skip i == 3
    }
    if i == 7 {
        break;      // stop at i == 7
    }
    print(i);       // prints 0, 1, 2, 4, 5, 6
}
```

## Functions

### Defining and Calling

```lion
func add(a, b) {
    return a + b;
}

print(add(3, 4));   // 7
```

Functions without a `return` statement return `nil`:

```lion
func greet(name) {
    print(f"Hello, {name}!");
}

let result = greet("Lion");   // prints "Hello, Lion!"
print(result);                 // nil
```

### Closures

Functions capture variables from their enclosing scope:

```lion
func make_counter(start) {
    let count = start;
    func inc() {
        count = count + 1;
        return count;
    }
    return inc;
}

let c = make_counter(0);
print(c());   // 1
print(c());   // 2
print(c());   // 3

// Each call creates an independent closure
let d = make_counter(100);
print(d());   // 101
print(c());   // 4 (c is independent)
```

### Anonymous Functions

```lion
let double = func(x) { return x * 2 };
print(double(5));   // 10
```

### Variadic Functions

Use `...` to accept any number of arguments:

```lion
func sum(...) {
    let total = 0;
    for n in args {
        total = total + n;
    }
    return total;
}

print(sum(1, 2, 3, 4));  // 10
```

The captured arguments are available as a list named `args` inside the function.

### Named Arguments

```lion
func draw_rect(width, height, color) {
    print(f"Drawing {color} rect {width}x{height}");
}

// Call with positional arguments
draw_rect(100, 50, "red");

// Call with named arguments (any order)
draw_rect(color = "blue", width = 200, height = 100);
```

You can mix positional and named arguments, but positional must come first.

### Recursion

```lion
func factorial(n) {
    if n <= 1 {
        return 1;
    }
    return n * factorial(n - 1);
}

print(factorial(5));  // 120
```

## Lambdas

Lambdas are anonymous functions with a concise syntax:

```lion
let double = |x| x * 2;
print(double(10));   // 20

// Multiple parameters
let add = |a, b| a + b;
print(add(3, 4));    // 7

// Multi-statement body
let clamp = |x, lo, hi| {
    if x < lo { return lo; }
    if x > hi { return hi; }
    return x;
};
```

Lambdas are especially useful with higher-order functions like `map`:

```lion
let nums = [1, 2, 3, 4, 5];
let doubled = map(nums, |x| x * 2);
print(doubled);   // [2, 4, 6, 8, 10]

let evens = filter(nums, |x| x % 2 == 0);
print(evens);     // [2, 4]
```

## Data Structures

### Lists

Ordered, zero-indexed, mutable collections:

```lion
let list = [1, 2, 3, 4, 5];

// Access elements
print(list[0]);      // 1 (first element)
print(list[-1]);     // 5 (last element — negative index)
print(list.len());   // 5

// Modify
list.push(6);        // append to end
let last = list.pop();  // remove and return last

// Iterate
for item in list {
    print(item);
}
```

### Dicts

Key-value maps. Keys can be strings, integers, or any hashable value.

```lion
let dict = {"a": 1, "b": 2, "c": 3};

// Access and modify
print(dict["a"]);         // 1
dict["d"] = 4;            // add or modify a key

// Check existence
print(dict.contains("a"));// true

// Get all keys
print(dict.keys());       // list of keys ["a", "b", "c", "d"]

// Loop over keys
for key in dict {
    print(key, dict[key]);
}
```

### Sets

Unordered collections of unique elements:

```lion
let s = {1, 2, 3};

s.insert(4);
print(s.contains(2));     // true
s.remove(2);
print(s);                 // {1, 3, 4}

for val in s {
    print(val);
}
```

### Tuples

Fixed-size, ordered collections. Tuples can hold values of different types.

```lion
let t = (1, "hello", 3.0);
print(t[0]);              // 1 (indexed access)
print(t[1]);              // "hello"

// Tuples are immutable — size and values cannot change
```

## Ranges

Ranges represent a sequence of integers from a start to an end (exclusive).

```lion
// Basic range: start..end (end is exclusive)
for i in 0..4 {
    print(i);   // 0, 1, 2, 3
}

// Ranges are values
let r = 1..10;
print(3 in r);  // true
print(0 in r);  // false

// Range with step: start..step..end
for i in 0..2..10 {
    print(i);   // 0, 2, 4, 6, 8
}

for i in 10..-3..0 {
    print(i);   // 10, 7, 4, 1
}
```

## Match Expressions

Match provides powerful pattern matching against literal values:

```lion
let x = 3;

match x {
    1 => print("one"),
    2 => print("two"),
    3 => {
        print("three");
    },
    _ => print("other"),    // default case
}
```

Match arms can be single expressions or blocks. The `_` wildcard matches anything.

## Error Handling

### Throwing Errors

```lion
throw "something went wrong";
throw f"invalid value: {x}";
```

### Catching Errors

```lion
try {
    let result = risky_operation();
    print(result);
} catch err {
    print(f"caught: {err}");
}
```

If no error occurs, the `catch` block is skipped. If an error is thrown inside `try`, execution jumps to `catch` with the error message bound to the variable.

## Modules and Imports

Lion has a built-in module system for organizing code.

### Importing Modules

```lion
// Import an entire module (access with module prefix)
import math;
print(math.sqrt(16));   // 4.0

// Import specific symbols
from math import sqrt, pow;
print(sqrt(16));        // 4.0

// Import with module alias
import math as m;
print(m.sqrt(16));

// Import symbol with alias
from math import sqrt as sq;
print(sq(16));
```

### Exporting Symbols

```lion
// Export a function
export func greet(name) {
    print(f"Hello, {name}!");
}

// Export a constant
export const PI = 3.14159;

// Export specific names
export { greet, PI };
```

## Full Example

Here's a complete program demonstrating closures, ranges, higher-order functions, and I/O:

```lion
func fibonacci(n) {
    if n <= 1 {
        return n;
    }
    return fibonacci(n - 1) + fibonacci(n - 2);
}

func main() {
    print("Fibonacci sequence:");
    for i in 0..10 {
        print(f"fib({i}) = {fibonacci(i)}");
    }

    let nums = [3, 7, 1, 9, 4];
    print(f"Sum: {sum(nums)}");

    let doubled = map(nums, |x| x * 2);
    print(f"Doubled: {doubled}");
}

func sum(list) {
    let total = 0;
    for n in list {
        total = total + n;
    }
    return total;
}

func map(list, fn) {
    let result = [];
    for item in list {
        result.push(fn(item));
    }
    return result;
}
```

Save this as `example.lion` and run:

```bash
lion run example.lion
```

## VS Code Setup

Lion provides a VS Code extension with syntax highlighting and a language server for diagnostics and completions.

### Installing the Extension

```bash
# 1. Build the LSP server
cargo build --bin lion-lsp

# 2. Install extension dependencies
cd vscode-lion
npm install
cd ..

# 3. Install the extension
code --install-extension vscode-lion/
```

If `code` is not on PATH, open VS Code, go to Extensions (Ctrl+Shift+X), click `...` → `Install from VSIX...` and select the `vscode-lion/` folder.

### LSP Features

The `lion-lsp` server provides:

- **Diagnostics** — syntax errors are underlined as you type
- **Completions** — keyword and built-in function suggestions
- **Hover info** — shows position info (line/column)

The LSP starts automatically when you open a `.lion` file in VS Code. It uses the same parser as the CLI, so error messages match exactly.

### Troubleshooting

| Problem | Fix |
|---------|-----|
| "lion-lsp not found" | Run `cargo build --bin lion-lsp` in the project root |
| No syntax highlighting | Open a `.lion` file and check the bottom-right shows "Lion" as the language mode |
| LSP not starting | Open the VS Code Output panel (Ctrl+Shift+U) and select "Lion Language Server" from the dropdown |
