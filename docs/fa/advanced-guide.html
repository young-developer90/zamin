راهنمای پیشرفته

این راهنما قابلیت‌های پیشرفته زبان زمین را پوشش می‌دهد شامل افزودن افزونه‌های C، تعامل با پایتون، استفاده از شتاب‌دهی GPU از طریق CUDA، جاسازی زمین در برنامه‌های Rust و استفاده از سرور LSP داخلی.

افزونه‌های C

زمین از افزونه‌های بومی C برای کدهای بحرانی از نظر عملکرد یا بسته‌بندی کتابخانه‌های C موجود پشتیبانی می‌کند. افزونه‌ها به صورت اشیاء مشترک کامپایل شده و به طور خودکار در زمان اجرا بارگذاری می‌شوند.

فایل هدر

هدر اصلی را در کد C خود اضافه کنید:

#include "zamin.h"

هدر در مسیر include/zamin.h در درخت منبع زمین قرار دارد.

انواع زمین

زمین ثابت‌های نوع C زیر را برای استفاده با اتحاد ZaminValue ارائه می‌دهد:

ثابتتوضیحات
ZAMIN_NILمقدار تهی
ZAMIN_INTعدد صحیح (64 بیتی علامت‌دار)
ZAMIN_FLOATاعشاری با دقت دوبرابر
ZAMIN_BOOLبولی درست/نادرست
ZAMIN_STRINGرشته تخصیص‌یافته در هیپ

ساختار ZaminValue

ساختار ZaminValue یک اتحاد برچسب‌دار است که تمام مقادیر قابل عبور از مرز C/زمین را نمایش می‌دهد:

typedef struct {
    int type;
    union {
        int64_t i;
        double f;
        int b;
        const char *s;
    } as;
} ZaminValue;

ساختار ZaminModuleFunc

هر تابع صادرشده توسط یک ورودی ZaminModuleFunc توصیف می‌شود:

typedef struct {
    const char *name;
    ZaminValue (*func)(int argc, ZaminValue *argv);
    int arity;  // -1 for variadic
} ZaminModuleFunc;

صدور یک ماژول

هر افزونه C باید یک تابع zamin_module_init صادر کند که یک آرایه ZaminModuleFunc برگرداند و تعداد را از طریق اشاره‌گر ارائه‌شده تنظیم کند:

 ZaminModuleFunc *zamin_module_init(int *count) {
    static ZaminModuleFunc funcs[] = { ... };
    *count = sizeof(funcs) / sizeof(funcs[0]);
    return funcs;
}

ساخت

افزونه خود را به عنوان یک کتابخانه مشترک کامپایل کنید:

gcc -shared -fPIC -O2 -I../include -o mymod.so mymod.c

فایل .so حاصل را در دایرکتوری modules/ پروژه خود قرار دهید. زمین به طور خودکار آن را هنگام نوشتن use mymod بارگذاری می‌کند.

مثال کامل: math_ext

#include "zamin.h"

static ZaminValue math_add(int argc, ZaminValue *argv) {
    if (argc != 2) {
        return (ZaminValue){ ZAMIN_NIL, { .i = 0 } };
    }
    int64_t a = argv[0].as.i;
    int64_t b = argv[1].as.i;
    return (ZaminValue){ ZAMIN_INT, { .i = a + b } };
}

static ZaminValue math_multiply(int argc, ZaminValue *argv) {
    if (argc != 2) {
        return (ZaminValue){ ZAMIN_NIL, { .i = 0 } };
    }
    int64_t a = argv[0].as.i;
    int64_t b = argv[1].as.i;
    return (ZaminValue){ ZAMIN_INT, { .i = a * b } };
}

ZaminModuleFunc *zamin_module_init(int *count) {
    static ZaminModuleFunc funcs[] = {
        { "add",      math_add,      2 },
        { "multiply",  math_multiply,  2 },
    };
    *count = sizeof(funcs) / sizeof(funcs[0]);
    return funcs;
}

استفاده در زمین:

use math_ext

let result = math_ext.add(10, 20)
print(result)  // 30

let product = math_ext.multiply(6, 7)
print(product)  // 42

تعامل با پایتون

زمین می‌تواند مستقیماً ماژول‌های پایتون را زمانی که با قابلیت python کامپایل شده باشد فراخوانی کند. این امکان استفاده مجدد از اکوسیستم گسترده پایتون را در کد زمین فراهم می‌کند.

ساخت با پشتیبانی پایتون

cargo build --features python

وارد کردن ماژول‌های پایتون

از py.import() برای آوردن یک ماژول پایتون به محدوده استفاده کنید:

let np = py.import("numpy")

let arr = np.array([1, 2, 3, 4, 5])
print(arr.mean())  // 3.0

بررسی نسخه پایتون

print(py.version())  // e.g. "3.14.0"

let requests = py.import("requests")

print(c)
// [[19 22]

شتاب‌دهی CUDA / GPU

زمین می‌تواند از GPUهای NVIDIA برای محاسبات زمانی که CUDA در دسترس است استفاده کند. سیستم ساخت به طور خودکار CUDA را در زمان کامپایل از طریق build.rs تشخیص می‌دهد.

تشخیص خودکار

در طول cargo build، build.rs کامپایلر nvcc و ابزارهای CUDA را بررسی می‌کند. اگر یافت شوند، ماژول‌های GPU به طور خودکار کامپایل و پیوند می‌شوند. نیازی به پرچم‌های اضافی نیست.

ماژول LINUM

ماژول LINUM (عددی زمین) primitives شبکه عصبی با شتاب GPU را فراهم می‌کند:

  • linum.tensor() - ایجاد تنسور GPU
  • linum.matmul() - ضرب ماتریس روی GPU
  • linum.relu() / linum.sigmoid() - توابع فعال‌سازی
  • linum.softmax() - Softmax برای طبقه‌بندی
  • linum.cross_entropy() - توابع خطا

API حافظه GPU

print(linum.gpu_name())        // "NVIDIA GeForce RTX 4090"
print(linum.gpu_memory())      // 24576 (MB)
print(linum.gpu_memory_used()) // 512 (MB)

Example: Neural Network

use linum

let model = linum.Sequential([
    linum.Linear(784, 256),
    linum.ReLU(),
    linum.Linear(256, 128),
    linum.ReLU(),
    linum.Linear(128, 10),
    linum.Softmax()
])

let optimizer = linum.Adam(model.parameters(), lr: 0.001)
let loss_fn = linum.CrossEntropyLoss()

for epoch in 0..10 {
    let pred = model.forward(train_data)
    let loss = loss_fn(pred, train_labels)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    print("Epoch {epoch}, Loss: {loss.item():.4f}")
}

جاسازی در Rust

زمین می‌تواند به عنوان یک کتابخانه در برنامه‌های Rust جاسازی شود. این برای ساخت ابزارها، IDEها یا برنامه‌هایی که به قابلیت اسکریپت‌نویسی نیاز دارند مفید است.

وابستگی Cargo.toml

[dependencies]
zamin = { path = "../zamin" }

API

دو تابع اصلی ارائه شده است:

  • zamin::execute_source(code: &str) - اجرای یک رشته کد زمین
  • zamin::execute_file(path: &str) - اجرای یک فایل زمین از دیسک

مثال کامل جاسازی در Rust

func main() {
    // Execute a Zamin source string
    zamin::execute_source(r#"
        let greeting = "Hello from Zamin!"
        print(greeting)
        let result = 2 + 3
        print("2 + 3 = {result}")
    "#).expect("Failed to execute Zamin source");

    // Execute a Zamin file
    zamin::execute_file("scripts/app.zamin").expect("Failed to execute file");

    // Access Zamin values from Rust
    let mut vm = zamin::Vm::new();
    vm.execute("let x = 42").unwrap();
    let value = vm.get_global("x").unwrap();
    println!("Zamin returned: {:?}", value);
}

سرور LSP

زمین با یک پیاده‌سازی داخلی از پروتکل سرور زبان (LSP) عرضه می‌شود که قابلیت‌های IDE را برای هر ویرایشگر با پشتیبانی LSP فراهم می‌کند.

فایل باینری zamin-lsp

پ از ساخت زمین، فایل باینری zamin-lsp در دایرکتوری خروجی در دسترس خواهد بود. این فایل از طریق stdio با استفاده از پروتکل LSP ارتباط برقرار می‌کند.

قابلیت‌ها

  • تکمیل خودکار - تکمیل خودکار آگاه از زمینه برای کلمات کلیدی، توابع داخلی، APIهای ماژول و متغیرهای تعریف‌شده توسط کاربر
  • هاور - اطلاعات نوع و مستندات هنگام هاور برای همه نمادها
  • تشخیص خطا - خطاهای نحوی بلادرنگ، ناهماهنگی نوع و مراجع تعریف‌نشده

افزونه VS Code

یک افزونه VS Code در دسترس است. آن را از بازارچه نصب کنید و موارد زیر را به settings.json خود اضافه کنید:

{
  "zamin.server.path": "/path/to/zamin-lsp"
}

این افزونه برجسته‌سازی نحوی، خط‌کشی خطا، پنجره‌های تکمیل خودکار و مستندات هاور را به صورت آماده فراهم می‌کند.
