مدیریت پروژه

زمین با یک زنجیره ابزار کامل CLI برای ایجاد، اجرا، فرمت‌دهی و تست پروژه‌ها ارائه می‌شود. این صفحه هر دستور و قالب مانیفست پروژه را پوشش می‌دهد.

دستورات CLI

دستورتوضیحات
zamin run <file> [--disassemble]اجرای یک فایل منبع زمین. از --disassemble برای چاپ بایت‌کد قبل از اجرا استفاده کنید.
zamin replشروع حلقه تعاملی خواندن-ارزیابی-چاپ (REPL).
zamin versionچاپ نسخه زمین، تاریخ ساخت و نسخه LLVM.
zamin fmt <file>فرمت‌دهی یک فایل منبع زمین در محل.
zamin test [filter]اجرای همه تست‌ها یا فقط تست‌های منطبق با filter.
zamin new <name>ایجاد ساختار دایرکتوری یک پروژه جدید با چیدمان استاندارد.
zamin initمقداردهی یک مانیفست zamin.json در دایرکتوری جاری.
zamin buildساخت پروژه به صورت یک فایل اجرایی مستقل.
zamin-rs <file>اجراگر سریع - کامپایل و اجرای یک فایل در یک مرحله.

ساختار پروژه

با دستور زیر یک پروژه جدید ایجاد کنید:

zamin new my_project

این دستور چیدمان زیر را تولید می‌کند:

my_project/
├── zamin.json
├── src/
│   └── main.zamin
├── tests/
│   └── main_test.zamin
└── modules/

همچنین می‌توانید یک مانیفست را در یک دایرکتوری موجود مقداردهی کنید:

mkdir my_project && cd my_project
zamin init

مانیفست zamin.json

فایل zamin.json پروژه شما را توصیف می‌کند. در اینجا قالب کامل با تمام فیلدهای پشتیبانی‌شده آمده است:

{
  "name": "my_project",
  "version": "1.0.0",
  "description": "A short description of the project.",
  "author": "Your Name",
  "license": "MIT",
  "entry": "src/main.zamin",
  "dependencies": {
    "http": "^1.0.0",
    "json": "^2.1.0"
  },
  "dev_dependencies": {
    "test_utils": "^0.5.0"
  },
  "scripts": {
    "start": "zamin run src/main.zamin",
    "test": "zamin test",
    "fmt": "zamin fmt src/"
  },
  "modules_path": "modules/",
  "target": "linux-x86_64",
  "optimization": "release"
}
فیلدنوعتوضیحات
namestringنام پروژه
versionstringنسخه معنایی
descriptionstringتوضیحات کوتاه
authorstringنام نویسنده
licensestringشناسه مجوز (SPDX)
entrystringفایل نقطه ورود (پیش‌فرض: src/main.zamin)
dependenciesobjectوابستگی‌های زمان اجرا
dev_dependenciesobjectوابستگی‌های فقط توسعه
scriptsobjectاسکریپت‌های نام‌گذاری شده قابل اجرا با zamin run --script <name>
modules_pathstringمسیر بارگذاری ماژول‌های بومی (پیش‌فرض: modules/)
targetstringسه‌تایی هدف ساخت
optimizationstring"debug" یا "release"

REPL

REPL تعاملی را با دستور زیر شروع کنید:

zamin repl

REPL از دستورات داخلی زیر پشتیبانی می‌کند:

دستورتوضیحات
exitخروج از REPL
helpنمایش دستورات و میانبرهای موجود
historyچاپ تاریخچه دستورات جلسه جاری

تکمیل خودکار با Tab

کلید Tab را فشار دهید تا کلمات کلیدی، توابع داخلی و شناسه‌های قبلاً استفاده شده به صورت خودکار تکمیل شوند. برای دیدن همه گزینه‌های ممکن، Tab را دو بار فشار دهید.

فرمت‌دهنده

زمین شامل یک فرمت‌دهنده کد داخلی است. از آن برای اعمال سبک یکسان استفاده کنید:

zamin fmt src/main.zamin

همچنین می‌توانید همه فایل‌های یک دایرکتوری را فرمت کنید:

zamin fmt src/

فرمت‌دهنده idempotent است - اجرای چندباره آن خروجی یکسانی تولید می‌کند. به تورفتگی موجود احترام می‌گذارد و ترتیب importها را تغییر نمی‌دهد.

تست

زمین یک چارچوب تست داخلی دارد. با استفاده از ماژول test تست بنویسید:

use test

func test_addition() {
    test.assert_eq(1 + 1, 2, "1 + 1 should equal 2");
    test.assert(2 + 2 == 4, "2 + 2 should equal 4");
}

func test_string_reverse() {
    let input = "hello";
    let reversed = input.reverse();
    test.assert_eq(reversed, "olleh", "reverse of 'hello' is 'olleh'");
}

func test_error_handling() {
    test.assert_raises(func() {
        let x = nil;
        x.value();
    }, "accessing nil should raise an error");
}

تست‌های خود را با دستور زیر اجرا کنید:

zamin test

فقط تست‌هایی که نامشان با یک فیلتر مطابقت دارد اجرا کنید:

zamin test addition

این دستور فقط test_addition را اجرا می‌کند. فیلتر به عنوان زیررشته در برابر همه نام‌های توابع تست تطبیق داده می‌شود.
