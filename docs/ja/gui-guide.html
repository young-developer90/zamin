GUI ツールキットガイド

Zamin のクロスプラットフォーム GUI ツールキットでネイティブデスクトップアプリケーションを構築。

概要

Zamin には 2 つの GUI ツールキットが付属しており、同じ API サーフェスを共有しているため、あるツールキット用に書かれたコードは変更なしで他のツールキットでも動作します。

  • Sol（Windows）— Win32 ベースのツールキットで、Windows では組み込みモジュールとして提供されます。外部依存関係は不要です。
  • Luna（Linux）— GTK4 ベースのツールキットで、luna フィーチャーフラグの背後で利用可能です。ホストシステムに GTK4 開発ライブラリが必要です。

どちらのツールキットも同じ関数名とウィジェットコンストラクタを公開しています。プラットフォームに合わせて選択するだけで、残りのコードは同じままです。

💡 注記

Windows では sol() が常に利用可能です。Linux では luna() を使用し、--features luna を付けてビルドします。

クイックスタート

以下は、ラベルを表示するウィンドウを表示する最小限の GUI アプリケーションです。

let gui = sol()  // or luna() on Linux
let win = gui.Leo("My App", 400, 300)
let label = gui.Label(win, "Hello, Zamin!")
gui.pack(label)
gui.mainloop(win)

これを app.zamin として保存して実行します。

$ zamin run app.zamin

タイトル "My App" のウィンドウが表示され、中央に "Hello, Zamin!" のテキストが表示されます。

ウィジェットリファレンス

Leo（ウィンドウ）

すべてのウィジェットを保持するトップレベルウィンドウ。

シグネチャ説明
Leo(title?, width?, height?) 新しいトップレベルウィンドウを作成します。width/height が省略された場合のデフォルトは 640×480 です。 
let gui = sol()
let win = gui.Leo("Settings", 800, 600)

Frame

子ウィジェットをグループ化して整理するためのコンテナウィジェット。

シグネチャ説明
Frame(parent) 指定された親ウィジェット内にフレームを作成します。 
let gui = sol()
let win = gui.Leo("App", 400, 300)
let frame = gui.Frame(win)
gui.pack(frame)

Label

静的テキストを表示します。ユーザー入力は受け付けません。

シグネチャ説明
Label(parent, text?) 親ウィジェット内に指定されたテキストのラベルを作成します。 
let gui = sol()
let win = gui.Leo("App", 400, 300)
let lbl = gui.Label(win, "Status: Ready")
gui.pack(lbl)

Button

クリック可能なボタン。押されるとコールバックがトリガーされます。

シグネチャ説明
Button(parent, text?, callback?) オプションのラベルとクリックハンドラを持つボタンを作成します。 
let gui = sol()
let win = gui.Leo("App", 400, 300)

func on_click() {
    print("Button was clicked!");
}

let btn = gui.Button(win, "Click Me", on_click)
gui.pack(btn)
gui.mainloop(win)

コールバックは通常の Zamin 関数です。引数を受け取ったり、ローカル状態をキャプチャするクロージャにすることもできます。

Entry

ユーザー入力のための単一行テキスト入力フィールド。

シグネチャ説明
Entry(parent) 親ウィジェット内にテキスト入力フィールドを作成します。 
let gui = sol()
let win = gui.Leo("App", 400, 300)
let entry = gui.Entry(win)
gui.pack(entry)

レイアウト管理

Zamin GUI ツールキットは、ウィジェットを配置するための 2 つのレイアウトメソッドを提供します。

pack

シグネチャ説明
pack(widget, side?, padx?, pady?) pack ジオメトリマネージャを使用してウィジェットを配置します。side は "top"、"bottom"、"left"、"right" のいずれかを指定できます。padx と pady は水平方向と垂直方向のパディングをピクセル単位で追加します。 
gui.pack(label, "top", 10, 5)
gui.pack(btn, "left", 5, 5)

place

シグネチャ説明
place(widget, x, y, w?, h?) ウィジェットを絶対座標で配置します。w と h で幅と高さを設定できます（オプション）。 
gui.place(label, 20, 50, 200, 30)

ウィジェット操作

これらの関数は、作成後にウィジェットを変更または照会します。

関数説明
config(widget, property, value) ウィジェットのプロパティを設定します。一般的なプロパティ："text"、"command"、"font"、"fg"、"bg"。
get(entry) Entry ウィジェットの現在のテキスト内容を返します。
insert(entry, pos, text) Entry ウィジェット内の指定された位置にテキストを挿入します。
delete(entry, start, end?) Entry ウィジェット内の start から end までの文字を削除します。end が省略された場合は 1 文字削除します。
title(window, text) ウィンドウのタイトルを変更します。
geometry(window, w, h) ウィンドウを指定された幅と高さにリサイズします。
destroy(widget) ウィジェットをレイアウトから削除し、そのリソースを解放します。
messagebox(text, title?) 指定されたテキストとオプションのタイトルでモーダルメッセージダイアログを表示します。 
gui.config(btn, "text", "Updated Label")
let value = gui.get(entry)
gui.title(win, "New Title")
gui.messagebox("Operation complete!", "Info")

例：電卓

表示フィールドと算術ボタンを備えた完全な電卓アプリケーション。

let gui = sol()
let win = gui.Leo("Calculator", 280, 350)

let display = gui.Entry(win)
gui.pack(display, "top", 10, 10)

let expression = ""

func on_digit(d) {
    expression = expression + d;
    gui.config(display, "text", expression);
}

func on_operator(op) {
    expression = expression + " " + op + " ";
    gui.config(display, "text", expression);
}

func on_clear() {
    expression = "";
    gui.config(display, "text", "");
}

func on_equal() {
    let result = eval(expression);
    gui.config(display, "text", str(result));
    expression = str(result);
}

// Build the button row for digits 1-9
for row in 0..3 {
    let frame = gui.Frame(win);
    gui.pack(frame, "top");
    for col in 0..3 {
        let digit = str(row * 3 + col + 1);
        let d = digit;
        let btn = gui.Button(frame, digit, func() { on_digit(d) });
        gui.pack(btn, "left", 2, 2);
    }
}

// Bottom row: 0, clear, equals
let bottom = gui.Frame(win)
gui.pack(bottom, "top")
let b0 = gui.Button(bottom, "0", func() { on_digit("0") })
gui.pack(b0, "left", 2, 2)
let bc = gui.Button(bottom, "C", on_clear)
gui.pack(bc, "left", 2, 2)
let be = gui.Button(bottom, "=", on_equal)
gui.pack(be, "left", 2, 2)

// Operator buttons
let ops = gui.Frame(win)
gui.pack(ops, "top")
for op in ["+", "-", "*", "/"] {
    let o = op;
    let b = gui.Button(ops, op, func() { on_operator(o) });
    gui.pack(b, "left", 2, 2);
}

gui.mainloop(win)

例：テキストエディタ

入力フィールド、開くボタン、保存ボタンを備えた基本的なテキストエディタ。

let gui = sol()
let win = gui.Leo("Text Editor", 600, 400)

// File path entry
let top_frame = gui.Frame(win)
gui.pack(top_frame, "top", 5, 5)

gui.Label(top_frame, "File:")
let path_entry = gui.Entry(top_frame)
gui.pack(path_entry, "left", 5, 5)

// Text area (simulated with a large Entry)
let text_area = gui.Entry(win)
gui.pack(text_area, "top", 10, 10)

// Button bar
let btn_frame = gui.Frame(win)
gui.pack(btn_frame, "top", 5, 5)

func on_open() {
    let path = gui.get(path_entry);
    if path == "" {
        gui.messagebox("Enter a file path first.", "Error");
        return;
    }
    let content = read(path);
    gui.config(text_area, "text", content);
    gui.title(win, "Text Editor - {path}");
}

func on_save() {
    let path = gui.get(path_entry);
    if path == "" {
        gui.messagebox("Enter a file path first.", "Error");
        return;
    }
    let content = gui.get(text_area);
    write(path, content);
    gui.messagebox("File saved!", "Success");
}

let open_btn = gui.Button(btn_frame, "Open", on_open)
gui.pack(open_btn, "left", 5, 5)

let save_btn = gui.Button(btn_frame, "Save", on_save)
gui.pack(save_btn, "left", 5, 5)

gui.mainloop(win)

プラットフォームに関する注意

Sol（Windows）

  • Win32 API 上に直接構築されており、外部依存関係やランタイムライブラリは不要です。
  • Windows 版 Zamin のビルドに組み込みモジュールとして同梱されています。
  • ネイティブウィジェットレンダリングによる標準 Windows のルックアンドフィールをサポートします。

Luna（Linux）

  • 最新の Linux デスクトップ環境向けに GTK4 上に構築されています。
  • GTK4 開発ライブラリのインストールが必要です。
# Debian / Ubuntu
sudo apt install libgtk-4-dev

# Fedora
sudo dnf install gtk4-devel

# Arch
sudo pacman -S gtk4
  • luna フィーチャーを有効にして Zamin をビルドします。
cargo build --release --features luna

macOS

  • macOS サポートは将来のリリースで計画されています。現時点の macOS では、標準ライブラリモジュールを使用してコマンドラインツールを構築できます。
✔ ヒント

Sol と Luna は同じ API を共有しているため、GUI コードを一度記述すれば、ファイルの先頭で適切なツールキットコンストラクタを選択するだけで、さまざまなプラットフォーム向けにコンパイルできます。
