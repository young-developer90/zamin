高度な機能ガイド

このガイドでは、C による Zamin の拡張、Python との相互運用、CUDA による GPU アクセラレーション、Rust アプリケーションへの Zamin の埋め込み、ビルトイン LSP サーバーの使用など、Zamin 言語の高度な機能について説明します。

C 拡張

Zamin は、パフォーマンスが重要なコードや既存の C ライブラリのラッピングのために、ネイティブ C 拡張をサポートしています。拡張は共有オブジェクトとしてコンパイルされ、実行時に自動的にロードされます。

ヘッダーファイル

C ソースにメインヘッダーをインクルードします：

#include "zamin.h"

ヘッダーは Zamin ソースツリーの include/zamin.h にあります。

Zamin の型

Zamin は ZaminValue 共用体で使用するために、以下の C 型定数を公開しています：

定数説明
ZAMIN_NILNil / null 値
ZAMIN_INT整数（64 ビット符号付き）
ZAMIN_FLOAT倍精度浮動小数点数
ZAMIN_BOOLブーリアン true/false
ZAMIN_STRINGヒープ割り当て文字列

ZaminValue 構造体

ZaminValue 構造体は、C/Zamin 境界を越えられるすべての値を表すタグ付き共用体です：

typedef struct {
    int type;
    union {
        int64_t i;
        double f;
        int b;
        const char *s;
    } as;
} ZaminValue;

ZaminModuleFunc 構造体

エクスポートされる各関数は ZaminModuleFunc エントリで記述されます：

typedef struct {
    const char *name;
    ZaminValue (*func)(int argc, ZaminValue *argv);
    int arity;  // -1 for variadic
} ZaminModuleFunc;

モジュールのエクスポート

すべての C 拡張は、ZaminModuleFunc 配列を返し、提供されたポインタを介してカウントを設定する zamin_module_init 関数をエクスポートする必要があります：

 ZaminModuleFunc *zamin_module_init(int *count) {
    static ZaminModuleFunc funcs[] = { ... };
    *count = sizeof(funcs) / sizeof(funcs[0]);
    return funcs;
}

ビルド

拡張を共有ライブラリとしてコンパイルします：

gcc -shared -fPIC -O2 -I../include -o mymod.so mymod.c

生成された .so ファイルをプロジェクトの modules/ ディレクトリに配置します。use mymod と書くと Zamin が自動的にロードします。

完全な例：math_ext

#include "zamin.h"

static ZaminValue math_add(int argc, ZaminValue *argv) {
    if (argc != 2) {
        return (ZaminValue){ ZAMIN_NIL, { .i = 0 } };
    }
    int64_t a = argv[0].as.i;
    int64_t b = argv[1].as.i;
    return (ZaminValue){ ZAMIN_INT, { .i = a + b } };
}

static ZaminValue math_multiply(int argc, ZaminValue *argv) {
    if (argc != 2) {
        return (ZaminValue){ ZAMIN_NIL, { .i = 0 } };
    }
    int64_t a = argv[0].as.i;
    int64_t b = argv[1].as.i;
    return (ZaminValue){ ZAMIN_INT, { .i = a * b } };
}

ZaminModuleFunc *zamin_module_init(int *count) {
    static ZaminModuleFunc funcs[] = {
        { "add",      math_add,      2 },
        { "multiply",  math_multiply,  2 },
    };
    *count = sizeof(funcs) / sizeof(funcs[0]);
    return funcs;
}

Zamin での使用：

use math_ext

let result = math_ext.add(10, 20)
print(result)  // 30

let product = math_ext.multiply(6, 7)
print(product)  // 42

Python 相互運用

Zamin は python 機能を有効にしてコンパイルすると、Python モジュールを直接呼び出せます。これにより、Zamin コードから広大な Python エコシステムを再利用できます。

Python サポート付きビルド

cargo build --features python

Python モジュールのインポート

py.import() を使用して Python モジュールをスコープに取り込みます：

let np = py.import("numpy")

let arr = np.array([1, 2, 3, 4, 5])
print(arr.mean())  // 3.0

Python バージョンの確認

print(py.version())  // e.g. "3.14.0"

型マーシャリング

Zamin は 2 つの言語間で自動的に型を変換します：

例

requests を使用して URL を取得：

let requests = py.import("requests")

let resp = requests.get("https://httpbin.org/json")
print(resp.status_code)
print(resp.json())

numpy を使用した行列演算：

let np = py.import("numpy")

let a = np.array([[1, 2], [3, 4]])
let b = np.array([[5, 6], [7, 8]])
let c = np.dot(a, b)
print(c)
// [[19 22]
//  [43 50]]

CUDA / GPU アクセラレーション

Zamin は CUDA が利用可能な場合、NVIDIA GPU を計算に活用できます。ビルドシステムは build.rs を介してコンパイル中に CUDA を自動検出します。

自動検出

cargo build 中に、build.rs が nvcc コンパイラと CUDA ツールキットを確認します。見つかった場合、GPU モジュールは自動的にコンパイルおよびリンクされます。追加のフラグは必要ありません。

LINUM モジュール

LINUM（Zamin Numerical）モジュールは、GPU アクセラレーションによるニューラルネットワークプリミティブを提供します：

  • linum.tensor() - GPU テンソルの作成
  • linum.matmul() - GPU での行列乗算
  • linum.relu() / linum.sigmoid() - 活性化関数
  • linum.softmax() - 分類用ソフトマックス
  • linum.cross_entropy() - 損失関数

GPU メモリ API

print(linum.gpu_name())        // "NVIDIA GeForce RTX 4090"
print(linum.gpu_memory())      // 24576 (MB)
print(linum.gpu_memory_used()) // 512 (MB)

例：ニューラルネットワーク

use linum

let model = linum.Sequential([
    linum.Linear(784, 256),
    linum.ReLU(),
    linum.Linear(256, 128),
    linum.ReLU(),
    linum.Linear(128, 10),
    linum.Softmax()
])

let optimizer = linum.Adam(model.parameters(), lr: 0.001)
let loss_fn = linum.CrossEntropyLoss()

for epoch in 0..10 {
    let pred = model.forward(train_data)
    let loss = loss_fn(pred, train_labels)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    print("Epoch {epoch}, Loss: {loss.item():.4f}")
}

Rust への埋め込み

Zamin はライブラリとして Rust アプリケーションに埋め込むことができます。これはツール、IDE、またはスクリプト機能が必要なアプリケーションの構築に役立ちます。

Cargo.toml 依存関係

[dependencies]
zamin = { path = "../zamin" }

API

2 つの主要な関数が提供されています：

  • zamin::execute_source(code: &str) - Zamin ソース文字列を実行
  • zamin::execute_file(path: &str) - ディスクから Zamin ファイルを実行

完全な Rust 埋め込み例

func main() {
    // Execute a Zamin source string
    zamin::execute_source(r#"
        let greeting = "Hello from Zamin!"
        print(greeting)
        let result = 2 + 3
        print("2 + 3 = {result}")
    "#).expect("Failed to execute Zamin source");

    // Execute a Zamin file
    zamin::execute_file("scripts/app.zamin").expect("Failed to execute file");

    // Access Zamin values from Rust
    let mut vm = zamin::Vm::new();
    vm.execute("let x = 42").unwrap();
    let value = vm.get_global("x").unwrap();
    println!("Zamin returned: {:?}", value);
}

LSP サーバー

Zamin にはビルトインの Language Server Protocol (LSP) 実装が含まれており、LSP をサポートする任意のエディタで IDE 機能を提供します。

zamin-lsp バイナリ

Zamin をビルドすると、zamin-lsp バイナリが出力ディレクトリに作成されます。これは LSP プロトコルを使用して stdio で通信します。

機能

  • 補完 - キーワード、組み込み関数、モジュール API、ユーザー定義変数のコンテキスト認識型自動補完
  • ホバー - すべてのシンボルに対する型情報とドキュメントのホバー表示
  • 診断 - リアルタイムの構文エラー、型の不一致、未定義参照

VS Code 拡張

VS Code 拡張が利用可能です。マーケットプレイスからインストールし、settings.json に以下を追加してください：

{
  "zamin.server.path": "/path/to/zamin-lsp"
}

拡張はシンタックスハイライト、エラー波線、自動補完ポップアップ、ホバードキュメントをすぐに提供します。
