プロジェクト管理

Zamin には、プロジェクトの作成、実行、フォーマット、テストのための完全な CLI ツールチェーンが付属しています。このページでは、すべてのコマンドとプロジェクトマニフェスト形式について説明します。

CLI コマンド

コマンド説明
zamin run <file> [--disassemble]Zamin ソースファイルを実行します。--disassemble を指定すると、実行前にバイトコードを表示します。
zamin replインタラクティブな REPL（Read-Eval-Print Loop）を起動します。
zamin versionZamin のバージョン、ビルド日、LLVM バージョンを表示します。
zamin fmt <file>Zamin ソースファイルをインプレースでフォーマットします。
zamin test [filter]すべてのテスト、または filter に一致するテストのみを実行します。
zamin new <name>標準レイアウトで新しいプロジェクトディレクトリをスキャフォールドします。
zamin initカレントディレクトリに zamin.json マニフェストを初期化します。
zamin buildプロジェクトをスタンドアロン実行可能ファイルにビルドします。
zamin-rs <file>クイックランナー - 1 つのファイルをコンパイルして 1 ステップで実行します。

プロジェクト構造

新しいプロジェクトを作成するには：

zamin new my_project

これにより、次のレイアウトが生成されます。

my_project/
├── zamin.json
├── src/
│   └── main.zamin
├── tests/
│   └── main_test.zamin
└── modules/

既存のディレクトリにマニフェストを初期化することもできます。

mkdir my_project && cd my_project
zamin init

zamin.json マニフェスト

zamin.json ファイルはプロジェクトを記述します。以下は、サポートされているすべてのフィールドを含む完全な形式です。

{
  "name": "my_project",
  "version": "1.0.0",
  "description": "A short description of the project.",
  "author": "Your Name",
  "license": "MIT",
  "entry": "src/main.zamin",
  "dependencies": {
    "http": "^1.0.0",
    "json": "^2.1.0"
  },
  "dev_dependencies": {
    "test_utils": "^0.5.0"
  },
  "scripts": {
    "start": "zamin run src/main.zamin",
    "test": "zamin test",
    "fmt": "zamin fmt src/"
  },
  "modules_path": "modules/",
  "target": "linux-x86_64",
  "optimization": "release"
}
フィールド型説明
namestringプロジェクト名
versionstringセマンティックバージョン
descriptionstring短い説明
authorstring作者名
licensestringライセンス識別子（SPDX）
entrystringエントリポイントファイル（デフォルト：src/main.zamin）
dependenciesobject実行時依存関係
dev_dependenciesobject開発時のみの依存関係
scriptsobjectzamin run --script <name> で実行可能な名前付きスクリプト
modules_pathstringネイティブモジュールをロードするパス（デフォルト：modules/）
targetstringビルドターゲットトリプル
optimizationstring"debug" または "release"

REPL

インタラクティブ REPL を起動するには：

zamin repl

REPL では次の組み込みコマンドが使用できます。

コマンド説明
exitREPL を終了
help使用可能なコマンドとショートカットを表示
history現在のセッションのコマンド履歴を表示

タブ補完

Tab キーを押すと、キーワード、組み込み関数、以前に使用した識別子が自動補完されます。Tab を 2 回押すと、すべての候補が表示されます。

フォーマッタ

Zamin には組み込みのコードフォーマッタが含まれています。一貫したスタイルを適用するには：

zamin fmt src/main.zamin

ディレクトリ内のすべてのファイルをフォーマットすることもできます。

zamin fmt src/

フォーマッタはべき等です。複数回実行しても同じ出力が得られます。既存のインデントは保持され、インポートの順序は変更されません。

テスト

Zamin には組み込みのテストフレームワークがあります。test モジュールを使用してテストを記述します。

use test

func test_addition() {
    test.assert_eq(1 + 1, 2, "1 + 1 should equal 2");
    test.assert(2 + 2 == 4, "2 + 2 should equal 4");
}

func test_string_reverse() {
    let input = "hello";
    let reversed = input.reverse();
    test.assert_eq(reversed, "olleh", "reverse of 'hello' is 'olleh'");
}

func test_error_handling() {
    test.assert_raises(func() {
        let x = nil;
        x.value();
    }, "accessing nil should raise an error");
}

テストを実行するには：

zamin test

名前がフィルタに一致するテストのみを実行するには：

zamin test addition

これにより test_addition のみが実行されます。フィルタはすべてのテスト関数名に対して部分一致でチェックされます。
