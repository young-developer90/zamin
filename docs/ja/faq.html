よくある質問

Zamin プログラミング言語に関するよくある質問と回答です。

Zamin をインストールするには？

Zamin はソースからビルドします（唯一のインストール方法です）：

git clone https://github.com/young-developer90/zamin.git
cd zamin
cargo build --release

バイナリは target/release/zamin にあります。PATH に追加してください。

文字列を連結するには？

+ 演算子（両方のオペランドが文字列である必要があります）または .. 演算子（文字列以外を自動変換）を使用します。

let greeting = "Hello" + " " + "World";
print(greeting);  // "Hello World"

// Use .. to coerce non-strings to strings
let n = 42;
print("The answer is " .. n);  // "The answer is 42"

// f-strings are preferred for interpolation
let name = "Alice";
print("Hello {name}");

let と const の違いは？

let は可変の変数を宣言します。const はコンパイル時に代入する必要がある不変の定数を宣言します。

let x = 10
x = 20  // OK

const PI = 3.14159
PI = 3.0  // Error: cannot reassign constant

ファイルを読み込むには？

fs 標準ライブラリモジュールを使用します：

let content = fs.read("data.txt");
print(content);

// Write to a file
fs.write("output.txt", "Hello, Zamin!");

HTTP リクエストを行うには？

http モジュールを使用します：

let resp = http.get("https://api.example.com/data");
print(resp.status);    // 200
print(resp.body);      // response body string

// POST with JSON body
let resp2 = http.post("https://api.example.com/submit",
    json.stringify({"name": "Alice", "email": "alice@example.com"})
);
print(resp2.status);

エラー処理はどうするの？

Zamin はエラー処理に try/catch を使用し、エラーを発生させるには throw を使用します：

try {
    let content = fs.read("nonexistent.txt");
} catch (e) {
    print(f"Error: {e}");
}

// Throw errors explicitly
func validate(n) {
    if n < 0 {
        throw Error("negative value not allowed");
    }
    return n;
}

try {
    validate(-1);
} catch (e) {
    print(f"Caught: {e}");
}

正規表現を使用するには？

re モジュールを使用します：

let text = "Contact: user@example.com, admin@test.com";

let first = re.find(r"[\w.]+@[\w.]+", text);
print(first);                          // "user@example.com"

let all = re.find_all(r"[\w.]+@[\w.]+", text);
print(all);                            // ["user@example.com", "admin@test.com"]

// Check if pattern matches
print(re.is_match(r"\d+", "abc 123"));  // true

// Replace matches
let cleaned = re.sub(r"\d+", "#", "abc 123 def 456");
print(cleaned);                         // "abc # def 456"

// Extract capture groups
let caps = re.captures(r"(\w+)@(\w+)", "user@example.com");
print(caps);                            // ["user@example.com", "user", "example.com"]

JSON をパースするには？

json モジュールを使用します：

let data = json.parse('{"name": "Alice", "age": 30}');
print(data["name"]);   // "Alice"
print(data["age"]);    // 30

// Serialize to JSON string
let obj = {"name": "Bob", "age": 25};
let str = json.stringify(obj);
print(str);             // {"name":"Bob","age":25}

// Pretty-print with indentation
print(json.pretty(obj));

プロジェクトを作成するには？

zamin new コマンドを使用します：

zamin new my_project
cd my_project
zamin run src/main.zamin

これにより、zamin.json、src/main.zamin、tests/、modules/ ディレクトリを含む標準のプロジェクトレイアウトが作成されます。

テストを実行するには？

test モジュールを使ってテスト関数を記述し、zamin test を実行します：

use test

func test_addition() {
    test.assert_eq(1 + 1, 2);
}

func test_string_upper() {
    test.assert_eq("hello".upper(), "HELLO");
}
zamin test

名前でテストをフィルタリング：zamin test addition。

Zamin はマルチスレッドをサポートしていますか？

Zamin はシングルスレッドの VM で、ファイバー（軽量コルーチン）による協調的並行処理を採用しています。各ファイバーは同じスレッド内で実行され、明示的に制御を譲るため、共有状態の同期問題なしに並行タスクスケジューリングが可能です。

CPU 負荷の高い処理には、subprocess モジュールを介して独立したプロセスを起動できます。マルチスレッドの Rust アプリケーションから Zamin を呼び出す場合は、Rust の埋め込み API を通じて真の並列処理が利用可能です。

Linux で GUI を使用するには？

Zamin には GTK4 を介して Linux で動作する組み込み GUI ツールキットが含まれています。必要なシステム依存関係をインストールしてください：

# Ubuntu/Debian
sudo apt install libgtk-4-dev

# Fedora
sudo dnf install gtk4-devel

その後、luna モジュールを使用します（--features luna 付きでビルド）：

let win = luna.Leo("My App", 800, 600);
let label = luna.Label(win, "Hello, Linux!");
luna.pack(label);
luna.mainloop(win);

詳細は GUI ガイド を参照してください。

OpenCV を使用するには？

Zamin は cv モジュールを通じて OpenCV をラップします。まず OpenCV をシステムにインストールし、次に：

use cv

let img = cv.imread("photo.jpg")
let gray = cv.cvtColor(img, cv.COLOR_BGR2GRAY)
let edges = cv.canny(gray, 50, 150)
cv.imwrite("edges.jpg", edges)
print("Saved edges.jpg")

詳細は OpenCV ガイド を参照してください。

プログラムが遅いのはなぜ？

一般的な原因と修正方法：

  • ループ内での文字列結合 — 文字列への繰り返しの + の代わりに、f-文字列または list.join() を使用してください。
  • グローバル変数へのアクセス — ローカル変数は約10倍高速です。タイトなループでグローバル変数にアクセスする場合は、ローカル変数にキャッシュしてください。
  • 繰り返しのリスト割り当て — list.new(capacity) を使用して事前に割り当ててください。
  • 最適化されていないビルド — AOT 最適化を適用したリリースビルドには zamin build を使用してください。
  • 最適でないアルゴリズム — 最初に zamin run --disassemble でプロファイリングしてバイトコードを確認し、アルゴリズムの改善を検討してください。

詳細なベンチマークと最適化テクニックについては、パフォーマンス のページを参照してください。

Zamin に貢献するには？

Zamin は MIT ライセンスのオープンソースです。貢献するには：

  • バグ報告 — github.com/young-developer90/zamin/issues で Issue を開いてください
  • コードの提出 — リポジトリをフォークし、フィーチャーブランチを作成し、プルリクエストを開いてください
  • ドキュメントの改善 — すべてのドキュメントは docs/ ディレクトリに HTML ファイルとしてあります
  • フィードバックの共有 — 機能リクエストや提案は Issue トラッカーで歓迎します

ビルド手順とプロジェクト構造については README を参照してください。

Rust アプリケーションに Zamin を埋め込むには？

Zamin は crates.io で公開されています。依存関係として追加し、埋め込み API を使用します：

// Cargo.toml
[dependencies]
zamin = "1.7.3"
// main.rs
func main() {
    zamin::execute_source(r#"
        print("Hello from embedded Zamin!");
    "#).unwrap();

    zamin::execute_file("scripts/plugin.zamin").unwrap();
}

完全な API リファレンスについては、Rust への埋め込み のセクションを参照してください。
